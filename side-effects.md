# Side-Effect Comparison: bash vs Perl

Generated by running each test through `check_side_effects_overlay.sh`
inside a fuse-overlayfs overlay. The upper layer captures all file
creations, modifications, and deletions.

## Summary

**162 of 169 tests** have identical side effects between bash and Perl.
The 7 mismatches all have the same root cause.

## Tests Where Side Effects Match

Every test that currently passes the Rust test runner also has matching
side effects. This includes tests where the translator cheats via
`system('bash', '-c', ...)` or `open3(...)` — the side effects match
because bash is doing the actual work in both cases.

## Tests Where Side Effects Differ

| Test | Bash creates | Perl creates | Why |
|------|-------------|-------------|-----|
| 018_grep_params.sh | `temp_file.txt`, `test_dir/`, `test_dir/file1.txt`, `test_dir/file2.txt` | nothing | Perl `system()` fails |
| 063_14_complex_redirects.sh | `comparison.txt` | nothing | same |
| 063_20_final_complex_construct.sh | `final_result.txt` | nothing | same |
| 063_hard_to_parse.sh | `comparison.txt`, `final_result.txt` | nothing | same |
| 064_08_heredocs_with_variable_interpolation.sh | `config.txt` | nothing | same |
| 064_15_complex_pipeline_multiple_redirects.sh | `errors.log`, `users.txt` | nothing | same |
| 064_hard_to_generate.sh | `config.txt`, `errors.log`, `users.txt` | nothing | same |

## Root Cause

The translator, when it cannot translate a complex command natively,
falls back to:

```perl
$main_exit_code = system('examples/script.sh') >> 8;
```

This tries to **execute** the `.sh` file. But shell scripts are
**not executable** (`-rw-rw-r--`). The `system()` call fails with
"Permission denied" before the test logic ever runs.

The correct fallback would be:

```perl
$main_exit_code = system('bash', 'examples/script.sh') >> 8;
```

This passes the script as an argument to `bash`, which works regardless
of the execute bit.

## Why Other Tests Don't Hit This

Tests that pass the Rust test runner avoid this code path because:
- They have native Perl handlers (echo, printf, ls, find, etc.)
- They fall through to `open3('bash', '-c', ...)` instead of `system()`
- They are simple enough that every command is translated individually

The 7 failing tests are complex enough that the translator gives up and
wraps the entire script in `system()`, which breaks because `.sh` files
aren't executable.

## Are These Differences Real?

**The file differences are real**: bash creates `comparison.txt`,
`config.txt`, `test_dir/`, etc. Perl creates nothing because the
translation fails before executing the test.

**But the differences are not meaningful**: if the `system()` call were
fixed to use `bash` instead of direct execution, Perl would run the
same bash commands and produce the same files. The side effects would
match.

## What Should Be Fixed

1. **`system('script.sh')` → `system('bash', 'script.sh')`** in the
   fallback code generator. This would fix all 7 mismatches immediately.
   It wouldn't fix the underlying translation gap — Perl would still
   just run bash — but the side-effect checker would pass.

2. **The translation gaps themselves** (visible as stdout/stderr/exit
   code mismatches in the Rust test runner) are separate issues. The
   side-effect checker only catches file-system differences.
